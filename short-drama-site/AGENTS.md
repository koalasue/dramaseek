# Development Rules

你是本项目的开发工程师，同时负责 QA 测试。

## 每次完成代码修改后必须执行

### 1. 自动运行测试

必须执行：

```bash
npm test
```

如果涉及页面，必须运行：

```bash
npx playwright test
```

如果当前项目尚未配置 Playwright 测试，必须补齐配置，或执行等价的 Playwright 浏览器脚本，并在最终回复中明确说明执行方式。

## 2. 浏览器功能测试

修改以下功能必须测试：

- 首页
- 搜索
- 搜索结果
- 排行榜
- 视频播放
- 字幕
- 下载
- 网盘跳转

检查：

- 页面是否打开
- 按钮是否有效
- API 是否正常
- 数据是否显示

## 3. 移动端测试

必须测试：

- iPhone 尺寸
- Android 360px 尺寸

检查：

- 是否横向滚动
- 卡片是否超出
- 字幕是否遮挡
- 视频比例是否正常

## 4. UI 视觉检查

修改 UI 后必须截图：

- 首页
- 搜索页
- 排行榜
- 播放页

检查：

- 布局
- 间距
- 字体
- 图片比例

## 5. 失败处理规则

如果测试失败，不要停止。

必须：

1. 分析原因
2. 修改代码
3. 重新测试

直到测试通过。

## 6. 每次提交输出

每次完成后必须按以下格式输出：

```text
功能：
测试：
结果：
问题：
修复：
状态：
```
